1 本機 Python SMTP 機制：硬編碼與直接認證

在 Python 腳本中，程式碼必須主動提供帳號與 16 位元密碼。學生要在程式中直接宣告此金鑰。請框出以下程式碼向學生示範：

🔑 Python 密碼宣告與連線區塊 
# 學生必須在此變數直接寫入本機硬編碼的 16 位的應用程式密碼
app_password = "abcd efgh ijkl mnop" 

server = smtplib.SMTP("smtp.gmail.com", 587)
server.starttls()  # 建立 TLS 加密通道
server.login(sender_email, app_password) # 使用金鑰完成身份認證
💡 教師教學點：此方法需要學生嚴格遵守資安，告知他們若將此程式碼上傳 GitHub，密碼將在一分鐘內被 Google 的自動掃描系統廢除並發出安全警告。

2 雲端 GAS 前後端分離機制：免密碼與部署隔離

第一次學習 GAS 最大的阻礙是「每次改內容都要重新部署 Web App」。 我們可以引入前後端分離觀念：後端 `Code.gs` 當作免更動的「寄信郵件伺服器」；前端 `setup.js` 則是學生修改收件人、主旨與 HTML 的操作台。

☁️ Code.gs (後端 API)

部署在 Google 雲端。只負責解析 Request，呼叫系統權限寄信。不需要寫任何密碼，部署一次就再也不需要修改。

💻 setup.js (前端資料端)

放在學生本機或 HTML 中。學生任意修改信件內容、收件人，按下執行後動態傳送至 API。修改此處完全不需重部署 GAS！

🚀 第一次部署 GAS 核心心法：

請帶領學生在 Google Apps Script 中貼上 Code.gs 的程式碼，並完成 **一次性部署**（取得 API 網址）。 此後，學生若想變更「信件主旨、內容或收件人」，只需要修改本機 setup.js 內的變數並呼叫即可，完全不需要再回到 GAS 網頁重新按部署更新！

請引導學生開啟 GAS 專案並貼入此程式碼。**教學亮點：** 程式碼完全沒有寫入硬編碼密碼！ 

/**
 * ☁️ Code.gs (API 轉介引擎)
 * 部署指引：
 * 1. 貼上程式碼後，點擊「部署」 > 「新增部署」
 * 2. 類型選擇「網頁應用程式 (Web App)」
 * 3. 執行身分選擇：「我」(您的 Gmail)
 * 4. 誰有權存取選擇：「任何人 (Anyone)」
 * 5. 授權驗證完成後，複製產生的「網頁應用程式網址」
 */

function doPost(e) {
  var corsHeader = "*"; // 允許跨網域存取
  
  try {
    // 解析來自前端 setup.js 傳過來的動態 JSON 資料
    var params = JSON.parse(e.postData.contents);
    var recipient = params.recipient;
    var subject = params.subject;
    var body = params.body;

    if (!recipient || !subject || !body) {
      throw new Error("發信失敗！缺少必要參數：recipient, subject 或 body");
    }

    // 🔒 安全機制：直接利用 Google 核心 OAuth 權限發信（完全免打應用程式密碼）
    GmailApp.sendEmail(recipient, subject, "", { htmlBody: body });

    return ContentService.createTextOutput(JSON.stringify({
      "status": "success", 
      "message": "郵件已透過 GAS 成功發送！"
    }))
    .setMimeType(ContentService.MimeType.JSON)
    .setHeaders({ "Access-Control-Allow-Origin": corsHeader });

  } catch (error) {
    return ContentService.createTextOutput(JSON.stringify({
      "status": "error",
      "message": error.toString()
    }))
    .setMimeType(ContentService.MimeType.JSON)
    .setHeaders({ "Access-Control-Allow-Origin": corsHeader });
  }
}

// 支援跨來源 CORS 預檢
function doOptions(e) {
  return ContentService.createTextOutput("")
    .setHeaders({
      "Access-Control-Allow-Origin": "*",
      "Access-Control-Allow-Methods": "POST, OPTIONS",
      "Access-Control-Allow-Headers": "Content-Type"
    });
}

這是前端/本機執行的 JavaScript 檔。**教學亮點：** 學生要修改的資料通通集中在這裡，直接由 JS 的 `fetch()` 送出。 

/**
 * 💻 setup.js (前端資料控制器)
 * 學生在此自由修改信件內容，點擊發信即可直接傳遞，不需重複部署 API
 */

// 1. 貼上部署 Code.gs 後得到的 Web App 網址
const GAS_WEBAPP_URL = "您的_GAS_網頁應用程式網址"; 

// 2. 🎯 在這裡任意修改您的信件資料
const emailData = {
  recipient: "recipient@example.com", 
  subject: "測試信：不用重新部署的 GAS 實作",
  body: "<h2>你好！</h2><p>這封信直接用 setup.js 裡的變數資料進行替換。</p>"
};

// 3. 呼叫發信引擎 (Fetch POST)
function sendMail() {
  if (GAS_WEBAPP_URL === "您的_GAS_網頁應用程式網址") {
    console.error("請先填入您部署 Code.gs 得到的 API 網址！");
    return;
  }

  fetch(GAS_WEBAPP_URL, {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify(emailData) // 將學生修改的資料封裝並送往雲端
  })
  .then(() => console.log("發送信號已發送！請至收件匣確認！"))
  .catch(err => console.error("發送發生異常：", err));
}

🎯 雲端定時發信：免開機、零電費

使用 Google Apps Script 的「時間驅動觸發器」。就算學生關閉電腦，Google 伺服器每天早上依然會準時被喚醒。

1. 操作步驟：如何在 GAS 控制台配置定時器
  1. 在 GAS 專案編輯器，點擊左側邊欄的 **「鬧鐘/觸發條件 (Triggers)」**。
  2. 點選右下角藍色的 **「新增觸發條件 (Add Trigger)」**。
  3. 設定：
    • 選擇執行的功能：sendDailyReport
    • 來源選： **「時間驅動 (Time-driven)」**
    • 類型選： **「日計時器 (Day timer)」**
