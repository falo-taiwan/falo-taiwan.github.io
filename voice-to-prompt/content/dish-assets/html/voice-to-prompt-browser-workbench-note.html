瀏覽器內轉錄到 Prompt 的教學型 MVP

版本：v102 (v1.02)。這份說明是 single-file HTML，可直接分享、展示或交給其他 AI 作為上下文。

核心流程 
語音 / 音檔
→ 瀏覽器內轉錄
→ 手動選擇來源
→ AI 或規則後處理
→ Prompt Template
→ 可執行 Prompt

專案定位

這不是完整語音助理，也不是商用品質 ASR 系統。它是一個教學展示頁，用來證明聲音可以進入 ETL 流程，最後變成可以交給 AI 使用的 Prompt。

兩層設計

上方：能力確定區

只做環境檢測、Chrome Built-in AI 與 Whisper 的模型下載狀態、模型暖身與能力說明。這一區的預設不是工作區的真正選擇。

下方：真正工作區

使用者用按鈕選已可用的 Whisper 模型、後處理模型與文字來源。工作區按鈕才決定實際流程。

轉錄模式

Web Speech API

  • 零安裝
  • 麥克風即時轉錄
  • 適合快速教學展示
  • 不能處理上傳音檔

Transformers.js Whisper

  • 瀏覽器內跑 Whisper
  • 可處理上傳音檔
  • 可用 chunk 做近即時轉錄
  • 首次需下載模型

模型選擇

Whisper 模型先在上方做預下載 / 暖身。已完成暖身的模型顯示為綠色，才可在工作區選擇；未下載模型顯示為灰色。MVP 建議先暖身 Xenova/whisper-tiny 或 Xenova/whisper-base。

AI 後處理

可選 Chrome Built-in AI、Ollama 或本地規則。Chrome Built-in AI 為預設選擇，目標語言預設繁中；Ollama 需檢查 endpoint 與模型清單，本地規則永遠可用。

v102 新增 Chrome Built-in AI 檢測與下載暖身：上方工具列只做檢測；下方狀態卡分別檢查 Summarizer、Language Detector、Translator、Rewriter 與 Prompt API / LanguageModel。Translator、Rewriter 與 Prompt API 可作為文字後處理候選；Summarizer 與 Language Detector 主要作為能力確認。狀態卡按鈕會透過 create({ monitor }) 觸發瀏覽器內建模型下載，並使用 downloadprogress 呈現進度。

模型暫存清理

頁面提供「清理模型暫存狀態」按鈕，用於釋放目前 Chrome Built-in AI session、清除本頁記錄的 Chrome AI / Whisper 可用狀態，並嘗試刪除 Transformers.js / Whisper 相關 Cache Storage 與 IndexedDB。Chrome Built-in AI 的實體模型由 Chrome 自動管理，網頁不能直接刪除；若要完整清理，需到 Chrome 設定或 chrome://on-device-internals 觀察與處理。

文字框操作

所有主要文字框都提供「清除」與「複製到剪貼簿」兩個按鈕，包含 Web Speech 輸出、Whisper 上傳輸出、Whisper 近即時輸出、待處理原始文字、修飾後文字與最終 Prompt。

Prompt DB

目前內建任務拆解、會議摘要、客戶需求整理、教材/文章發想、待辦事項產生五類模板。
