⚡ 填單自動化：特殊控制與 Vue/React 狀態同步之本質

💡 研發背景與實戰定位： 本專案（FALO ERP Form Helper）是專為網頁 DOM 操作、狀態同步、資料儲存與跨宇宙 Message Passing 通訊設計的企業自動化實戰教案。我們將解析如何透過外掛對瀏覽器頁面上的輸入框、按鈕與文字標籤進行「強行控制與狀態更新」之黑科技。

在開發企業內部 ERP 自動化時，最常見的問題是：網頁欄位看似被填入了資料，但是點擊儲存時卻「被清空」或「提示必填欄位為空」。這是因為現代前端框架（React/Vue/Angular）具有嚴密的單向或雙向 Reactivity 狀態管理。以下為您揭露本專案突破前端框架限制的四項核心技術：

🛡️ 技術一：Vue/React Reactivity 狀態同步 (Event Dispatching)

開發痛點： 當 Content Script 單純使用 input.value = "newValue" 時，只更改了 DOM 底層的 value 屬性，並沒有觸發 Vue 的 v-model 或 React 的 onChange 監聽，導致框架內部的資料狀態依然是空值。

解決方案： 外掛在寫入 value 之後，會主動發送 Event 廣播，通知框架將該元素視為「已被使用者手動觸發變更」： 

input.value = value;
input.dispatchEvent(new Event('input', { bubbles: true }));
input.dispatchEvent(new Event('change', { bubbles: true }));

加上 bubbles: true 能確保事件往上傳遞，觸發 Vue 或 React 在上層元件註冊的全域監聽，完美同步前端狀態。

🎨 技術二：動態 CSS 閃爍提示注入 (CSS Flash Animation)

開發痛點： 外掛在全自動填單時，使用者無法直觀確認哪些欄位是被外掛填寫的、哪些填寫失敗，降低了對自動化軟體的信任感。

解決方案： 外掛透過 content.css 注入了特殊的黃色閃爍動畫，並在 content.js 填值時動態為該輸入框加上 class，並在動畫結束後自動移除： 

@keyframes flash-yellow {
  0% { background-color: #fef08a; }
  100% { background-color: #ffffff; }
}
.flash-active {
  animation: flash-yellow 1s ease-out;
}

這為使用者提供了極佳的視覺反饋，能直觀感受到自動填單的精準與動感。

⚙️ 技術三：自動化點擊控制鏈 (Remote Click Chain)

開發痛點： ERP 頁面中，點擊「確認入庫」前必須先點擊「檢核表單」。如果在檢核成功前，外掛強行點擊「確認入庫」，會因為按鈕是 disabled 而無效，必須做狀態檢測或延時等待。

解決方案： 藉由 Content Script 偵測網頁狀態，點擊檢核按鈕後，利用 `setTimeout` 延遲 500ms 等待 DOM 更新，再偵測入庫按鈕是否啟用，隨後模擬點擊，實現多按鈕順序遙控點擊鏈。 

document.querySelector('#btn-check-form').click();
setTimeout(() => {
  const submitBtn = document.querySelector('#btn-submit-entry');
  if (!submitBtn.disabled) {
    submitBtn.click();
  }
}, 500);

🔄 技術四：BOM CSV 處理與 FileReader 還原

開發痛點： 當匯出 CSV 時，Excel 開啟常會將 UTF-8 編碼誤判為 ANSI，導致繁體中文欄位全是亂碼。匯入時則需要純前端 FileReader 解析。

解決方案： 匯出時，在 CSV 純文字資料前加上「BOM (Byte Order Mark)」位元組：\uFEFF，強制 Excel 用 UTF-8 來讀取；匯入時，以 FileReader 非同步讀取檔案文字： 

// 匯出加 BOM
const blob = new Blob(['\uFEFF' + csvText], { type: 'text/csv;charset=utf-8;' });
// 匯入 FileReader
const reader = new FileReader();
reader.onload = (e) => { parseCSV(e.target.result); };
reader.readAsText(file);

🧩 Chrome 擴充功能 MV3 核心架構解析

Chrome 外掛程式的本質是瀏覽器中的「輕量級網頁應用」。在最新的 Manifest V3 (MV3) 規範中，擴充功能主要由不同執行環境的元件組成。理解這些元件的權限與邊界，是與 AI 協同開發時的核心基本功。

🛡️ Chrome 外掛必備三劍客（核心必備）

- manifest.json (身分設定清單)：外掛的註冊檔，聲明外掛資訊、圖示、以及要求 activeTab 與 sidePanel 安全權限，並宣告 background.js 與 content.js 的注入路徑。

- background.js (背景守護程序)：全域事件處理程序（Service Worker）。本專案中監聽 Icon 點擊事件，並指令開啟右側 Side Panel 側邊欄。

- content.js / content.css (網頁注入爪牙)：直接注入當前 ERP 網頁 DOM 中運行，擁有直接控制網頁欄位填值與執行黃色閃爍特效的權限。

🔌 本專案額外補充組件（高階補充）

- sidepanel.html / sidepanel.js (側邊欄控制台)：外掛側邊欄的 HTML 面板與控制腳本。運行於獨立 Extension 宇宙，實作欄位隨機產生、CSV 備份與還原、遠端點擊遙控與流水號捕捉報告下載，透過 Message Passing 與 content.js 通訊。

- erp.html (外掛演練靶機)：本次課程的模擬 ERP 表單網頁，具備欄位檢核、儲存入庫與流水序號機制，提供給外掛進行雙向讀寫控制演練。
