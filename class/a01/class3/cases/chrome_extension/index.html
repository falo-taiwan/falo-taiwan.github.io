Chrome Extension DOM Automation

AI 時代的外掛控制實戰：
Chrome Extension 填單助手

本案將帶您透過實作，深入理解：如何寫瀏覽器外掛來控制網頁 DOM 節點，實現表單自動化。

傳統上，網頁自動化或自動表單填寫常需使用複雜的 RPA 工具。但現在，透過開發一個輕量級的 Chrome 外掛，我們就能利用簡單的 JS DOM 操作與通訊協定，實現強大的網頁填單助手。

在進入外掛控制前，我們在課堂開頭先下載並安裝了 FALO Instagram Video Enhancer (v1.0.1) 擴充功能，體驗了外掛如何注入程式碼。接著，我們將延伸此概念：如果我們是要「控制網頁、幫忙進行 ERP 表單自動化」呢？本案將帶您透過實作，直觀理解 Chrome 外掛與網頁 DOM 的通訊控制。

外掛雙向控制核心觀念剖析

寫入網頁 (Write)
外掛 ➔ 網頁
外掛側邊欄的 JavaScript 向 active tab 發送指令與數據，網頁端的 content.js 接收到後，透過尋找 CSS Selector 將數值填入對應的 DOM 輸入框中。
擷取資料 (Read)
網頁 ➔ 外掛
當需要讀取網頁狀態時，外掛發送擷取訊息，注入網頁的 content.js 讀取指定 DOM 欄位的 value，並透過 sendResponse 跨視窗回傳填入外掛控制項。

V1：Chrome Extension (幫人操作)

我們實作的第一個階段是 **ERP 表單輔助器**。當我們載入一個 Chrome Extension 到網頁時，外掛主要進行雙向資料控制：

  • 讀取網頁 (Read DOM)：檢測頁面上的輸入欄位與系統狀態，並讀取 DOM 節點的 value 值回傳至側邊欄中。
  • 修改網頁 (Write DOM)：在外掛側邊欄發送寫入指令，將數值填寫進對應的網頁 DOM 輸入框中，填值成功後伴隨亮黃色的填值閃爍視覺反饋。

這代表 Extension 可以完全控制網頁，但是它有一個技術特性與維護要點 (需注意事項)：外掛高度依賴網頁的 DOM 結構。只要前端代碼重構、CSS Selector 改變，外掛對應的定位邏輯就需要同步維護與更新，否則將會找不到目標元素而失效。

📡 實體測試網頁 DOM 欄位規格對齊 (DOM Selector Specifications)

外掛之所以能精準控制網頁填單，是因為外掛開發與網頁結構之間遵守同一個 **DOM 規範契約**。當外掛被載入至我們的 實體外掛測試頁 (erp.html) 後，對齊的 DOM 規格定義如下，學員可在此對照網頁元件的定位資訊：

元件名稱 CSS Selector (ID) 資料型態與說明
採購單號輸入框 #po-number String 類型的文字輸入框
產品代碼輸入框 #product-code String 類型的文字輸入框
進貨部門選單 #dept-code select 元素，選項包含 PUR, WH, QC, RD
進貨數量輸入框 #received-qty Number 類型的數值輸入框
產品效期輸入框 #expiry-date Date 類型的日期選擇輸入框

檢核表單按鈕 #btn-check-form 主網頁上用來驗證欄位是否非空與格式的按鈕
確認入庫按鈕 #btn-submit-entry 主網頁上辦理正式資料寫入的入庫按鈕

🔌 Chrome 擴充功能元件劃分與架構剖析

一個完整的瀏覽器自動化外掛，是由 Chrome 必備的核心設定檔與專案專屬 UI 共同協作完成。其功能與角色定位如下：

🛡️ Chrome 外掛必備三劍客（必備黨）

這是 Chrome 外掛得以加載並運行的**最基本身分與核心權限檔案**。缺一不可，是擴充功能的骨架。

  • manifest.json 擴充功能身分證
    外掛的主設定檔（JSON）。聲明外掛名稱、版本、權限（如 activeTab、sidePanel）、背景 Service Worker 以及要注入的內容腳本。瀏覽器全靠它來讀取擴充功能的規則。
  • background.js 後台守護程序
    事件驅動的背景 Service Worker 腳本。它不在網頁內運行，也無法碰觸網頁 DOM，但負責全局調度。本專案中負責監聽點擊外掛 Icon 的事件，並命令瀏覽器開啟右側 Side Panel 面板。
  • content.js 網頁注入爪牙
    唯一**直接注入到當前開啟網頁（Tab 宇宙）**中執行的 JavaScript 程式。它擁有直接讀寫網頁 DOM 節點的權力。本專案中負責執行「填值、派遣 input/change 事件以相容 Vue/React、讓網頁欄位黃色閃爍、回傳流水號與遙控點擊按鈕」。
🔌 本專案專屬補充說明（額外組件）

本專案為了提供強大的 **ERP 表單自動填寫 UI 與資料備份** 所額外擴充的組件，負責呈現介面與實作高級輔助功能。

  • sidepanel.html & sidepanel.js 控制台 UI 與邏輯
    外掛側邊欄的面板頁面與控制腳本。運行在獨立的 Extension 宇宙中，無法直接存取網頁 DOM，因此必須與 content.js 進行訊息通訊。這裡實作了隨機產生、CSV 匯出匯入、遙控控制與生成下載入庫報告。
  • content.css 動態樣式注入
    隨 content.js 一起注入到目標網頁的 CSS 樣式。專門用來宣告閃爍的 CSS 動畫（例如 `faloHighlightYellow` 與 `faloHighlightPink`），讓 content.js 能為輸入框動態加上閃爍類別。
  • icon.png 外掛圖示
    外掛在瀏覽器工具列上所顯示的視覺標誌，也是點擊啟動側邊欄的按鈕。
  • erp.html (外掛演練靶機) 課程專屬網頁
    **本次課程的實體測試網頁**。模擬企業 ERP 的採購單表單（具備欄位檢核、儲存入庫以及動態流水序號生成）。外掛即是在這張網頁上進行讀寫控制演練。

📡 跨宇宙雙向通訊（Message Passing）架構圖

由於外掛側邊欄（Side Panel）與當前網頁（Tab DOM）屬於不同的執行空間（宇宙），兩者要完成控制，必須透過 `chrome.tabs.sendMessage` 與 `chrome.runtime.onMessage` 來互寄信件：

附錄：V1 網頁外掛核心程式碼剖析

此處列出 V1 階段注入網頁的 Content Script 核心邏輯。學習者可以從代碼中看出「外掛如何透過遍歷 DOM 來理解網頁結構」： 

// 遍歷 DOM，篩選頁面上所有的可輸入元素
function getFormFields() {
  return Array.from(
    document.querySelectorAll(
      'input:not([type="submit"]):not([type="hidden"]), select, textarea'
    )
  );
}

// 根據 DOM 階層與 ID/Name 屬性，計算出唯一的 CSS Selector 路徑
function getCSSSelector(el) {
  if (el.id) return `#${el.id}`;
  if (el.name) return `${el.tagName.toLowerCase()}[name="${el.name}"]`;
  
  // 向上遞歸父層
  const parent = el.parentElement;
  if (parent && parent.id) {
    return `#${parent.id} > ${el.tagName.toLowerCase()}`;
  }
  return el.tagName.toLowerCase();
}
