Chrome Extension DOM Automation

AI 時代的外掛控制實戰：
Chrome Extension 填單助手

本案將帶您透過實作，深入理解：如何寫瀏覽器外掛來控制網頁 DOM 節點，實現表單自動化。

傳統上，網頁自動化或自動表單填寫常需使用複雜的 RPA 工具。但現在，透過開發一個輕量級的 Chrome 外掛，我們就能利用簡單的 JS DOM 操作與通訊協定，實現強大的網頁填單助手。

在進入外掛控制前，我們在課堂開頭先下載並安裝了 FALO Instagram Video Enhancer (v1.0.1) 擴充功能，體驗了外掛如何注入程式碼。接著，我們將延伸此概念：如果我們是要「控制網頁、幫忙進行 ERP 表單自動化」呢？本案將帶您透過實作，直觀理解 Chrome 外掛與網頁 DOM 的通訊控制。

外掛雙向控制核心觀念剖析

寫入網頁 (Write)
外掛 ➔ 網頁
外掛側邊欄的 JavaScript 向 active tab 發送指令與數據，網頁端的 content.js 接收到後，透過尋找 CSS Selector 將數值填入對應的 DOM 輸入框中。
擷取資料 (Read)
網頁 ➔ 外掛
當需要讀取網頁狀態時，外掛發送擷取訊息，注入網頁的 content.js 讀取指定 DOM 欄位的 value，並透過 sendResponse 跨視窗回傳填入外掛控制項。

V1：Chrome Extension (幫人操作)

我們實作的第一個階段是 **ERP 表單輔助器**。當我們載入一個 Chrome Extension 到任意網頁時，外掛主要做了兩件事：

  • 讀取網頁 (Read DOM)：檢測頁面上的輸入欄位，計算總數，並把欄位唯一的 CSS Selector 找出來，產生網頁元素結構。
  • 修改網頁 (Write DOM)：在必填欄位加上粉紅色的磨砂玻璃發光外框，並提供「一鍵智慧填入測試資料」，直接修改節點的值。

這代表 Extension 可以完全控制網頁，但是它有一個重大致命傷：只要前端代碼重構、CSS Selector 改變，外掛的自動化腳本就會立刻崩潰。

📡 外掛與網頁 DOM 控制規範對齊 (DOM Selector Mapping)

外掛之所以能精準控制網頁填單，是因為外掛開發與網頁結構之間必須嚴格遵守同一個 **DOM 規範契約**。當外掛被載入至我們的 實體外掛測試頁 (erp.html) 後，外掛的 content.js 會透過以下規格對齊控制：

  • 採購單號輸入框： 網頁 ID 為 #po-number，外掛寫入對應的 po 欄位值。
  • 產品代碼輸入框： 網頁 ID 為 #product-code，外掛寫入對應的 code 欄位值。
  • 進貨數量輸入框： 網頁 ID 為 #received-qty，外掛寫入對應的 qty 欄位值。
  • 產品效期輸入框： 網頁 ID 為 #expiry-date，外掛寫入對應的 expiry 欄位值。
  • 現代框架防錯： 在 DOM 填值後，外掛必須手動派遣 input 與 change 事件，以防止現代網頁框架（React/Vue）狀態丟失。

💡 V1 實戰 Prompt 模擬：表單輔助開發

要做出表單輔助器，以下是我們指令的兩個階段：

💬 V1.1 基礎口語意圖 Prompt (V1-v1)
我想要寫一個 Chrome 外掛，當我進入 ERP 網頁時，可以自動幫我把所有的輸入框偵測出來，在必填欄位旁邊加上高亮標示，並且加一個按鈕讓我點一下就能把測試資料填進去。
⚡ V1.2 進階技術控制 Prompt (V1-v2)
請建立一個 Manifest V3 規格的 Chrome 外掛。 編寫 content.js，在網頁加載時以 document.querySelectorAll 篩選所有未隱藏的 input, select, textarea。 在 required 的欄位加上 glows-pink 的 CSS border highlighters。 在網頁右上角動態 inject 一個玻璃質感的 sidebar 面板，列出偵測到的所有欄位標籤與其唯一 CSS selector，並提供 'Focus' 與 'Smart Fill' 觸發事件。
🧠 檢視 AI Agent 內部的自主剖析與隱性執行工作 (V1)
  • 自動生成 Manifest V3 規格：解析開發者意圖，自動編寫 background.js 與 content_scripts 宣告。
  • 處理框架相容性 (Event Dispatching)：由於現代 ERP 可能使用 Vue 或 React，AI 自動在 content.js 中對填入的值調用 dispatchEvent(new Event('input')) 以觸發雙向綁定。
  • 防禦性 CSS 注入：為避免 host 網頁 CSS 污染，AI 在 content_scripts 裡特別宣告 CSS 權重 !important 隔離 sidebar 樣式。

附錄：V1 網頁外掛核心程式碼剖析

此處列出 V1 階段注入網頁的 Content Script 核心邏輯。學習者可以從代碼中看出「外掛如何透過遍歷 DOM 來理解網頁結構」： 

// 遍歷 DOM，篩選頁面上所有的可輸入元素
function getFormFields() {
  return Array.from(
    document.querySelectorAll(
      'input:not([type="submit"]):not([type="hidden"]), select, textarea'
    )
  );
}

// 根據 DOM 階層與 ID/Name 屬性，計算出唯一的 CSS Selector 路徑
function getCSSSelector(el) {
  if (el.id) return `#${el.id}`;
  if (el.name) return `${el.tagName.toLowerCase()}[name="${el.name}"]`;
  
  // 向上遞歸父層
  const parent = el.parentElement;
  if (parent && parent.id) {
    return `#${parent.id} > ${el.tagName.toLowerCase()}`;
  }
  return el.tagName.toLowerCase();
}
