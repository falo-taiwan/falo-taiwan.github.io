The Evolution of AI Web Automation

AI 控制數位世界的演化：
從 Chrome Extension 到 Computer Use

這不是在教寫 Chrome 外掛，而是在教：人如何控制網頁 ➔ AI 如何控制網頁。

傳統上，人們將網頁自動化或自動表單填寫歸類為「軟體工程」或「寫瀏覽器外掛」。但隨著 AI 技術發展，我們發現：Chrome Extension、Computer Use 與 AI Agent，其實都處在同一條「利用 AI 控制數位世界」的演化路線上。

在進入網頁控制演化前，我們在課堂開頭先下載並安裝了 FALO Instagram Video Enhancer (v1.0.1) 擴充功能，體驗了外掛如何透過注入代碼改善第三方影片播放體驗。接著，我們將延伸此概念：如果我們不是要「改善瀏覽體驗」，而是要「控制網頁、幫忙進行表單自動化」呢？本案將帶您透過實作，直觀理解此演化路徑：從「寫瀏覽器外掛來讀改網頁 DOM 節點 (V1)」，再到「利用大語言模型看螢幕、動滑鼠以操作任何系統 (V2)」。

Force 教學金句觀念剖析

V1 Version
幫人操作
Chrome Extension 依賴特定的瀏覽器權限，注入程式碼去讀取 DOM (文件物件模型) 結構。它能高亮、提醒、自動填入數值，在人類按下按鈕的輔助下完成流程。
V2 Version
AI 幫人操作
Computer Use 跨越了 DOM 節點的限制。AI 像人一樣「看著螢幕（Vision）」、「定位按鈕座標（Coordinates）」並「移動真實滑鼠點擊」，自主完成整套跨應用程式工作流。

V1：Chrome Extension (幫人操作)

我們實作的第一個階段是 **ERP 表單輔助器**。當我們載入一個 Chrome Extension 到任意網頁時，外掛主要做了兩件事：

  • 讀取網頁 (Read DOM)：檢測頁面上的輸入欄位，計算總數，並把欄位唯一的 CSS Selector 找出來，產生網頁元素結構。
  • 修改網頁 (Write DOM)：在必填欄位加上粉紅色的磨砂玻璃發光外框，並提供「一鍵智慧填入測試資料」，直接修改節點的值。

這代表 Extension 可以完全控制網頁，但是它有一個重大致命傷：只要前端代碼重構、CSS Selector 改變，外掛的自動化腳本就會立刻崩潰。

📡 外掛與網頁 DOM 控制規範對齊 (DOM Selector Mapping)

外掛之所以能精準控制網頁填單，是因為外掛開發與網頁結構之間必須嚴格遵守同一個 **DOM 規範契約**。當外掛被載入至我們的 實體外掛測試頁 (erp.html) 後，外掛的 content.js 會透過以下規格對齊控制：

  • 採購單號輸入框： 網頁 ID 為 #po-number，外掛寫入對應的 po 欄位值。
  • 產品代碼輸入框： 網頁 ID 為 #product-code，外掛寫入對應的 code 欄位值。
  • 進貨數量輸入框： 網頁 ID 為 #received-qty，外掛寫入對應的 qty 欄位值。
  • 產品效期輸入框： 網頁 ID 為 #expiry-date，外掛寫入對應的 expiry 欄位值。
  • 現代框架防錯： 在 DOM 填值後，外掛必須手動派遣 input 與 change 事件，以防止現代網頁框架（React/Vue）狀態丟失。

💡 V1 實戰 Prompt 模擬：表單輔助開發

要做出表單輔助器，以下是我們指令的兩個階段：

💬 V1.1 基礎口語意圖 Prompt (V1-v1)
我想要寫一個 Chrome 外掛，當我進入 ERP 網頁時，可以自動幫我把所有的輸入框偵測出來，在必填欄位旁邊加上高亮標示，並且加一個按鈕讓我點一下就能把測試資料填進去。
⚡ V1.2 進階技術控制 Prompt (V1-v2)
請建立一個 Manifest V3 規格的 Chrome 外掛。 編寫 content.js，在網頁加載時以 document.querySelectorAll 篩選所有未隱藏的 input, select, textarea。 在 required 的欄位加上 glows-pink 的 CSS border highlighters。 在網頁右上角動態 inject 一個玻璃質感的 sidebar 面板，列出偵測到的所有欄位標籤與其唯一 CSS selector，並提供 'Focus' 與 'Smart Fill' 觸發事件。
🧠 檢視 AI Agent 內部的自主剖析與隱性執行工作 (V1)
  • 自動生成 Manifest V3 規格：解析開發者意圖，自動編寫 background.js 與 content_scripts 宣告。
  • 處理框架相容性 (Event Dispatching)：由於現代 ERP 可能使用 Vue 或 React，AI 自動在 content.js 中對填入的值調用 dispatchEvent(new Event('input')) 以觸發雙向綁定。
  • 防禦性 CSS 注入：為避免 host 網頁 CSS 污染，AI 在 content_scripts 裡特別宣告 CSS 權重 !important 隔離 sidebar 樣式。

🤖 AI Agent 實戰：一鍵生成外掛 Prompt 指南

當我們想要寫一個 Chrome 外掛時，傳統上我們需要手動建立多個檔案、串接背景腳本與內容注入腳本，並處理複雜的跨視窗信件通訊。但在 AI Agent（如 Google Antigravity 或 Claude）的時代，我們可以直接給予 結構化提示詞 (Prompt)，讓 AI Agent 自主剖析環境、建立檔案並完成編寫。

💡 如何寫好一個給 AI Agent 的外掛開發 Prompt？

一個能讓 AI Agent 一鍵完美生成整個 Chrome 外掛的提示詞，必須具備以下 4 個核心要素：

1. 規格與版本限制： 明確指定使用 Manifest V3 規格，避免 AI 使用過時的 V2 語法。
2. 檔案架構清單： 表列所有需要建立的檔案名稱及其職責，這能引導 AI Agent 精準配置工具（Tool Call）來建立多檔案。
3. 框架相容與防錯細節： 指出目標網頁可能使用 React 或 Vue，要求 AI 在寫入 DOM 值後主動派發 input 與 change 事件。
4. 教學與版權約束： 要求在程式碼中加上詳盡註解及特定的版權水印（如 Falo x Force Cheng 2026/6/6），以利後續的程式碼追蹤與學員臨摹。

V2：Computer Use (AI 幫人操作)

如果在 V1 階段，我們更進一步問：「如果按按鈕、填表單的不是人，而是 AI 呢？」

這就帶出了 V2：Computer Use。在 Computer Use 的世界中，AI 不需要知道什麼 CSS Selector，也不管網頁是用 HTML 寫的、還是 Flash 甚至是一張靜態圖。AI Agent 透過以下循環操作：

V2 電腦操作核心迴圈：
1. 觀察畫面 (Vision) ➔ 讀取螢幕截圖。
2. 規劃決策 (Plan) ➔ 辨識目標按鈕在畫面的 (X, Y) 像素座標點。
3. 執行動作 (Action) ➔ 移動真實/虛擬滑鼠到 (X, Y) 進行點擊，或向輸入框發送鍵盤事件 (Keystrokes)。
4. 驗證結果 (Verify) ➔ 截取下一張圖，判斷是否成功，繼續下一步。

這種方式讓 AI 可以進行高複雜度的跨軟體整合流，例如： 登入銀河軟體 ➔ 開啟會計傳票新增頁 ➔ 依序填入日期、科目、摘要與借貸金額 ➔ 校對借貸平衡 ➔ 點擊儲存寫入總帳 ➔ 完成。

💡 V2 實戰 Prompt 模擬：跨軟體流程自動化

當我們希望 Agent 執行 Computer Use 自動化流程時的指令對照：

💬 V2.1 基礎口語意圖 Prompt (V2-v1)
幫我登入銀河軟體，把今天的會計傳票登錄進去，科目是應收帳款，摘要寫 6 月銷貨款項，借方和貸方都是 5 萬元，填完確認有平衡就存檔。
⚡ V2.2 進階技術控制 Prompt (V2-v2)
請啟動 OSWorld 環境與 Vision Agent。 步驟 1：在 Chrome 瀏覽器截圖中尋找 '銀河軟體' 登入按鈕，定位座標 (X, Y) 點擊並輸入帳密。 步驟 2：偵測並定位會計傳票欄位，自動填入當日日期、科目『1203 應收帳款』、摘要『6 月份銷貨款項』。 步驟 3：定位借方與貸方輸入框填入 50000 元，校對借貸平衡狀態。 步驟 4：定位『儲存會計傳票』按鈕並點擊，確認彈出儲存成功 JV 序號。
🧠 檢視 AI Agent 內部的自主剖析與隱性執行工作 (V2)
  • 畫面語意分割 (Semantic Segmentation)：在 Vision Transformer (ViT) 中對畫面進行像素分類，偵測按鈕文字，避開廣告或無關連結。
  • 座標換算與滑鼠抖動 (Coordinates Mapping)：將 OS 物理解析度（如 1920x1080）與瀏覽器內 viewport 縮放比進行校對，確保滑鼠指針精確定位到按鈕中央。
  • 非同步狀態驗證 (Reactivity Validation)：讀取借方與貸方輸入框內的數值，透過數學邏輯驗證借貸方金額是否相等，確認平衡狀態標示為 Balanced 後才執行儲存。

附錄：V1 網頁外掛核心程式碼剖析

此處列出 V1 階段注入網頁的 Content Script 核心邏輯。學習者可以從代碼中看出「外掛如何透過遍歷 DOM 來理解網頁結構」： 

// 遍歷 DOM，篩選頁面上所有的可輸入元素
function getFormFields() {
  return Array.from(
    document.querySelectorAll(
      'input:not([type="submit"]):not([type="hidden"]), select, textarea'
    )
  );
}

// 根據 DOM 階層與 ID/Name 屬性，計算出唯一的 CSS Selector 路徑
function getCSSSelector(el) {
  if (el.id) return `#${el.id}`;
  if (el.name) return `${el.tagName.toLowerCase()}[name="${el.name}"]`;
  
  // 向上遞歸父層
  const parent = el.parentElement;
  if (parent && parent.id) {
    return `#${parent.id} > ${el.tagName.toLowerCase()}`;
  }
  return el.tagName.toLowerCase();
}
