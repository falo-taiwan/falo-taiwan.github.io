🛠️ 外掛與網頁 DOM 互動通訊規範 (API & Selector Specifications) ▼ 展開檢視

當您將外掛安裝於 Chrome 並造訪此網頁時，外掛與本網頁是透過以下 DOM 節點規範與跨視窗訊息協議來進行互動填值的。這也是實作網頁自動化的底層標準：

欄位名稱 DOM Selector 資料格式 外掛通訊 Key
採購單號 #po-number String (PO-YYYY-XXXX) po
產品代碼 #product-code String (FALO-XX) code
進貨數量 #received-qty Number qty
產品效期 #expiry-date Date (YYYY-MM-DD) expiry
⚠️ 現代網頁框架 (React/Vue/Angular) 事件派遣防錯規範：
直接以 input.value = value 寫入值後，必須手動派遣 input 與 change 事件。否則，前端框架的 Virtual DOM 無法捕捉到狀態更新，在提交表單時會導致資料丟失： 
input.dispatchEvent(new Event('input', { bubbles: true }));
input.dispatchEvent(new Event('change', { bubbles: true }));
📡 跨分頁訊息通訊規範 (Extension Message Spec)：
外掛側邊欄發送指令至 Content Script 時，必須遵循以下格式： 
// 高亮指示
chrome.tabs.sendMessage(tabId, { action: "toggleHighlight" });
// 填寫指示
chrome.tabs.sendMessage(tabId, { action: "fillForm", data: { po, code, qty, expiry } });
