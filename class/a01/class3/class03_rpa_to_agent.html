主題十：從 ERP 外掛到 AI Agent

很多企業其實早已經站在 AI Agent 的前一站：理清企業自動化演進與治理核心觀念

本章定位

許多人在第一次接觸 AI Agent（如 Codex、Antigravity 等）時，容易誤以為這是一個全新的世界。但對於 ERP 顧問與系統開發人員而言，其實並非如此。因為**許多 ERP 客製化開發與外掛程式，本質上就是 Workflow Automation (工作流自動化) 與 RPA (機器人流程自動化) 的應用。** 本章旨在梳理這條技術演進路線，幫助學員看清流程本質。

銀河顧問最熟悉的 Workflow 模式

日常 ERP 系統開發中，以下三種經典客製需求在架構上都有共通的核心代碼模式：

#1 訂單信用檢查
訂單建立 ➔ 查額度 ➔ 寄送通知
訂單儲存時觸發 API 檢查客戶剩餘信用額度，若額度超限，則透過 LINE/Email 警示業務人員。
#2 採購審批拋轉
請購核准 ➔ 建採購單 ➔ 通知人員
請購單完成最終核准後，系統自動於資料庫拋轉建立採購單，並推送即時通知給採購擔當。
#3 安全存量警示
庫存過低 ➔ 觸發警示 ➔ 通知主管
當夜間批次盤點發現某原料低於安全存量，觸發預存程序 (SP) 寄送庫存預警給資材主管。

共通的核心代碼模式：Event (事件) ➔ Rule (規則) ➔ Action (行動) = Workflow (工作流) 的本質

ERP 外掛到 AI Agent 的六個演進階段

人工處理
Stage 1
高度依賴人力，進行打字輸入、查詢與跨表拋轉。工作重複性高、容易出錯且難以追蹤。
核心： 重複性高、易出錯、無效率
ERP 客製程式
Stage 2
利用預存程序 (SP)、資料庫 Trigger 或 SDK/外掛包，在 ERP 內部完成防呆、防錯與欄位自動拋轉。
技術： Trigger / SP / SDK 外掛
RPA 模擬
Stage 3
讓系統代替人員執行重複性的電腦桌面操作。自動登入網銀、下載報表、搬運資料。
代表： UiPath / Power Automate
工作流自動化
Stage 4
不再模仿人點擊畫面，而是直接串接 API、資料庫與雲端服務，進行純 JSON 的資料處理與自動流動。
代表： n8n / Make / Zapier
AI Agent 時代
Stage 5
引入大模型的理解與決策能力。具備理解需求、判斷、生成內容並自主呼叫工具 (Tool Use) 的能力。
最新案例：新旗艦 Claude Fable 5 幫助 Stripe 將數月專案縮減至數天完成，證實了 Agent 時代的效率躍升。
代表： Antigravity / Codex
Multi-Agent 網路
Stage 6
多個具備不同專長角色（如財務、庫存、採購 Agent）的代理人，在治理框架下自主分工與協作。
未來： 自主分工與網絡化協作

推薦白箱架構 n8n 的真實原因

Force 推薦 n8n，並非因為它開源免費，而是因為它屬於「白箱（White Box）架構」：

  • 資料流向完全透明： Workflow 的步驟、內嵌 Script、API Request/Response 在畫面上清晰可追溯。
  • 對人類與 AI 雙重友善：
    • 對於人類：交接時不需翻閱厚重 SOP，看圖即懂，利於系統維護與交接。
    • 對於 AI Agent：AI 可以直接讀取 n8n 的工作流 JSON 定義檔，進行分析、安全稽核、或自動生成新節點。

AI 時代的流程治理新觀念：AI Review Workflow

在 AI 時代，流程控制與稽核思維正在升級：

過去：人 Review Code ➔ 現在：AI Review Code ➔ 未來：AI Review Workflow

AI 將協同人類，共同審查企業自動化工作流中的四個核心維度：

1
流程合理性
檢查是否存在死鎖、無限循環或邏輯缺口。
2
權限管控
防範 API 越權呼叫，把關數據安全邊界。
3
例外處理
遇到斷線、鎖定時，是否有 HITL 人工介入。
4
流程冗餘
分析是否有重複流程與資安洩漏隱憂。

從 AI Governance 角度來看：白箱流程的優勢在於容易稽核、容易追溯、容易交接、容易維護。同時這也大幅方便了 AI Code Review、AI Workflow Review 與 AI Security Review。

"很多企業認為自己還沒接觸 AI Agent。但實際上，只要您做過 ERP 客製化、流程簽核、自動通知、資料交換，就已經站在 AI Agent 的前一站了。本章的重點不是學會某個特定工具，而是理清 ERP 外掛到 Multi-Agent 的上演進路線，為 Class 03 後續的《AI 與 AI 的協作》與《Multi-Agent 工作模式》打下最穩固的觀念地基。"
—— Force 觀點
