1. 多人共用多帳號下的 AI 協作模式

核心思維：帳號分離與開發解耦

在企業團隊協作與多人開發的情境下，我們提倡「記憶留在個人帳號，開發純開發」的核心原則。

同仁使用個人專屬的 ChatGPT 或 Claude 帳號進行邏輯推導、思路整理與 Prompt 微調，將個人對話脈絡與記憶保留在個人端；而在本地開發時，則將開發環境解耦，僅由個人帳號作為調動（Trigger）媒介，去喚醒專案目錄下專屬的 AI Agent（如 Antigravity / Codex）來執行具體開發工作，確保團隊共享程式庫的乾淨與專一性。

2. Gemini_Canvas 常見困擾與解法

突破 Canvas 限制與 PoC 避坑指南

Gemini Canvas 具備極強的視覺原型設計 (PoC) 能力，但同仁在實戰中常遇到 Context 丟失、程式碼截斷、以及前後文漂移等痛點。本章引導同仁建立防禦性 Prompt 設計，掌握如何對 AI 進行精準的程式碼區塊重新生成與約束。

4. md / html 雙軌文檔策略

知識資產化與開發軌跡治理
