## 1. 案例緣起：社群使用者的真實痛點

在許多 AI 學習與行銷社群中，使用者經常需要大量轉發相同的廣告文宣、活動通知或推廣訊息。然而，LINE 官方為了打擊垃圾訊息，採用了嚴格的**「精確匹配過濾 (Exact String Matching)」**與**「內容雜湊值 (Hash)」**比對機制。
* **使用者困擾**：短時間內向多個群組發送相同字串，會被 LINE 伺服器判定為機器人洗版，導致帳號遭到限制發言或永久封鎖。
* **解決思路**：在不影響人類視覺閱讀的前提下，動態改變每條訊息的底層編碼（雜湊值），或透過 AI 進行語意改寫，使伺服器認為每條訊息都是「獨一無二」的。

---

## 2. 第二階段：使用 Gemini Canvas 進行極簡 PoC 驗證

為了解決這個社群痛點，第一步是進行**概念驗證 (Proof of Concept, PoC)**：
* **開發工具**：使用 **Gemini Canvas** 模式。
* **實作內容**：快速編寫一個單一 HTML 檔案的極簡版本，包含基本輸入框、一個「零寬度字元 (Zero-Width Space)」插入按鈕，以及簡易的複製功能。
* **社群分享**：將此 PoC 工具分享給社群，快速獲得使用者回饋，證實「在字詞間隨機插入隱形 Unicode 字元（如 `\u200B`）」確實能有效突破 LINE 雜湊值偵測，且不影響視覺排版。

---

## 3. 第三階段：使用 Google Antigravity 高規重構進化

在 PoC 獲得成功後，為了提供更完整、更具商業感且能處理複雜業務的工具，我們使用 **Google Antigravity** 代理進行高規重構：
* **Glassmorphism 視覺美學**：採用 Outfit/Inter/Noto Sans TC 字型，搭配深色科技感毛玻璃背景與漸層發光（Glow Orbs），極具現代感。
* **參數微調與多重防護機制**：
  * **零寬度字元 (ZWS)**：在字詞間隨機安插。
  * **同形字替換 (Homoglyph)**：將英數字轉為全形或替換為俄文/希臘變體字（如 `c` ➔ `с`）。
  * **標點符號隨機變異**：隨機替換全半形符號，打破規則比對。
  * **交織中點 (Interleaving)**：在詞彙間插入微小中點打碎特徵。
  * **動態秒級時間戳記**：保證每條訊息的 100% 物理唯一性。
* **本地狀態持久化 (Persistence)**：使用 `localStorage` 自動保存使用者的 API Key、自訂問候語池及選項設定，提升使用者體驗。

## 4. 第四階段：代理化呈現（文件變形輸出與 Computer Use 模擬）

這是本案例最核心的**代理人思維 (Agentic Thinking)**，展示了 AI 時代下「資訊呈現」與「操作示範」的變革：

### A. 文件變形輸出 (Document Transformation)
系統不僅能生成變體，還支援：
* **零寬度字元視覺化開關**：允許人類一鍵「看見」隱形的字元，提升對工具黑盒子的理解。
* **結構化資料導出**：支援一鍵將生成的變體打包為 **CSV 格式**，包含「變體編號、字數、相似度、變體文字」，方便結合機器人進行自動化發送。

### B. 動態式 SOP：Computer Use 智能示範沙盒
傳統的產品教學需要長篇大論的圖文或錄影，而在本專案中，我們實作了 **Computer Use 模擬沙盒**：
1. **虛擬游標 (Virtual Cursor)**：在網頁上生成一個虛擬滑鼠指針，模擬 Agent 的移動路徑。
2. **模擬操作 (Action Emulation)**：點擊「開始示範」後，游標會自動移動到輸入框打字、拖曳變體數量 Slider、切換 AI/非 AI 模式頁籤，並點擊生成按鈕。
3. **終端日誌 (HUD Terminal)**：右側沙盒會即時輸出 `Thought: [思考步驟]`、`Tool Call: [呼叫工具]`、`Result: [執行結果]` 的 Console 日誌，向使用者直觀展示 AI Agent 自動執行任務的完整軌跡。

> [!TIP]
> **教學啟示：SOP 的「動態可視化」**
> 未來的軟體不再只是靜態工具，而是「AI 代理可以自動操作的系統」。本案例展示了**「代理人友善介面 (Agent-Friendly UI)」**與**「操作動態播放」**的設計理念，幫助學員理解如何將複雜的軟體操作指南，轉化為 AI Agent 可直接播放、展示的「動態 SOP」。
