AI 程式開發與原型代理 (Coding & Prototyping)

Codex / Antigravity Active Agent
具備地端終端指令執行與精密檔案 Diff 補丁修改能力的 AI 代理人。
實務重點：自動化執行規劃、執行與驗收三部曲。在修改代碼時應指示其使用 replace_file_content 進行局部 Diff 補丁，避免整檔覆寫浪費 Token 與引入額外 Bug。
Gemini Canvas 模式 Interactive Edit
網頁端雙向編輯視窗，利於代碼局部微調與文案編排。
常見困擾與解法：
  • 困擾一：對話框命名均為英文，難以管理。
    ➔ 解法：在開始時在提示詞規定命名，或手動重新命名為清晰的中文專案標籤。
  • 困擾二：思考與註解全為英文。
    ➔ 解法：在對話起點注入 Prompt 強制要求使用繁體中文（zh-TW）註解與思考。
Claude Code & Claude Fable 5 CLI & Flagship Model
Anthropic 的 Terminal 命令行代碼代理人，搭載最新發布的 Claude Fable 5（首度向企業公開 Mythos 級技術）。
實務重點： Fable 5 模型專攻高難度工程與因果推理，Stripe 實測顯示原本需數月的工程專案，透過 Fable 5 可縮短至數天內初步建置完畢。使用 Claude Code 能在 CLI 中直接對本地代碼進行重構、跑測試並自動生成 Git Commit 資訊。

雙軌文檔策略與部署 Prompt (Doc Strategy & Prompting)

md / html 雙軌策略 Memory vs Visual
同時兼顧 AI 檢索效率與人類學員閱讀體驗的雙向策略。
實務重點：
  • .md (AI 專用語音)：去除所有裝飾樣式，用簡潔的結構化 Markdown 作為 AI 的外部記憶體，節省 Token 並防範幻覺。
  • .html (人類專用介面)：使用 Premium RWD、OLED 亮暗主題切換器與毛玻璃質感，提供最高質感的視覺體驗。
本機 & GitHub Pages Prompt 策略 Path & GEO Adaptation
網頁檔案在不同發布環境下的路徑轉換與資訊標記。
實務重點：
  • 本機 Prompt：路徑使用相對路徑（如 ../../../），且外掛 API 應做 localStorage 降級以防網頁崩潰。
  • GitHub Pages Prompt：上線前要求 AI 將連結修正（如 ../../../），並強制要求加入防偽浮水印（Falo x Force Cheng 2026/6/8）與 GEO 搜尋爬蟲標記。

地雲整合與通信橋樑 (Integration & Bridging)

地端 Server (Python 呼叫) Local execution
透過本機 Python 伺服器（Flask/FastAPI），讓 HTML 前端與地端進行通訊。
實務重點：前端網頁使用 Fetch API 傳送 HTTP 請求給 localhost 的 Python，藉以執行本機的 shell 腳本、OCR 大宗分揀程式或檔案讀寫，達到地端自動化。
GAS 雲端 Serverless 範例 Google Cloud API
Google 輕量 Serverless API 後端，零成本快速與 Google Sheet、Gmail 對接。
實務重點：網頁發送 Fetch 請求給 GAS 部署的 Web App，GAS 接收後透過 API 寫入 Google 試算表（自動記帳），或呼叫 MailApp.sendEmail 進行信件寄送，完美實現雲端整合。
ngrok / LocalTunnel (穿透調試) Tunneling
地端與公網 Webhook 的通訊映射工具，利於本地即時除錯。
實務重點：將本機運行的埠（如 5000 埠）直接映射成帶有安全證書的 HTTPS 公網網址，便於直接在本地接收 LINE Bot 的 Webhook 即時事件，不需要上傳雲端即可本地中斷點 Debug。
