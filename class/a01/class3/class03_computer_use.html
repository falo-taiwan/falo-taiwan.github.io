Course Module 3.6

主題六：從社群痛點到 AI 代理協作：LINE 防封鎖工具與 Computer Use 模擬實務

剖析一個實用防封鎖工具從 PoC 到高規進化，再到 Computer Use 模擬操作之完整知識工程歷程

🚀 點此開啟 LINE 防封鎖工具實作項目

工具開發之進化四部曲

本案例的誕生展示了標準的「痛點導向 AI 協作開發流程」。我們從社群的真實問題出發，利用快速驗證工具（Gemini Canvas）搭建概念，隨後以 AI Agent（Google Antigravity）進行完整重構，最終以 Computer Use 實現自動化演示。

01
社群真實痛點
AI 行銷社群使用者反映頻繁轉發推廣訊息會被 LINE 判定洗版並封鎖帳號。其背後機制是 LINE 的 完全匹配比對 (Exact Matching) 與 雜湊值 (Hash) 比對。
02
Gemini Canvas 極簡 PoC
第一步使用 Gemini Canvas 快速產出單一 HTML 的極簡 PoC，測試並分享「在字詞間隨機插入隱形 Unicode 零寬度字元 (`\u200B`)」以改變雜湊值的方案，成功通過驗證。
03
Antigravity 高規進化
交由 Google Antigravity 進行生產級重構：導入漸層毛玻璃美學、全半形變異、同形字替換（Homoglyph）、自訂雞湯詞庫、`localStorage` 本地狀態保存與 API Key 安全儲存。
04
SOP 代理化呈現
實作文件變形輸出（隱形字元一鍵顯形、CSV 打包下載），以及 Computer Use 智能示範沙盒，模擬虛擬游標打字與點擊，以動態日誌向使用者展示自動化操作。

核心技術原理剖析

混合式防禦架構：大語言模型 + 傳統編碼算法

單純依賴 AI 改寫成本較高，且可能受限於速率限制。本工具實作了混合式防禦 (Hybrid Architecture)：傳統編碼算法負責「毫秒級、無上限的結構特徵破壞」，Gemini AI 則負責「高維度的語意同義改寫」。

  • 零寬度字元 (ZWS)：隨機安插 \u200B（零寬空格）與 \u200C（零寬不連字）。雖然人眼看不見，但每一處插入都會使字串的二進制 Hash 值徹底改變。
  • 同形異義字 (Homoglyph)：將一般英文字元（如 Latin a）替換為視覺外觀一模一樣的斯拉夫語系字元（如 Cyrillic а），在極端情況下可繞過特徵比對。
  • 秒級時間戳與問候語交織：在首尾動態加上毫秒級時間代碼，提供物理層面的 100% 訊息唯一性。

SOP 變革：Computer Use 示範沙盒的實踐

在未來的 AI 代理時代，軟體的使用手冊不再是靜態的文字，而是**「代理人可以自我示範並運作的操作指南」**。本專案在 script.js 中實作了一個極為精緻的 Computer Use 模擬器：

  • 游標定位動畫 (moveCursorToElement)：利用 CSS transition 與 getBoundingClientRect()，動態計算網頁中任何 DOM 元素（如按鈕、輸入框）的核心座標，將虛擬鼠標指針平滑地滑動至目標位置，創造極佳的 Agentic UX。
  • 模擬打字引擎 (typeTextSimulation)：利用 setTimeout 遞迴，設定每 30ms 派發一次打字事件，並主動觸發 input 與 change 事件，與網頁的 React/Vanilla JS 雙向綁定邏輯無縫對接。
  • 思考與工具日誌 HUD：在模擬過程中，沙盒的 HUD Console 會即時列印出： 
    Thought: 我需要批次生成 5 個變體訊息，我將切換到傳統演算法標籤...
Tool Call: hover_and_click("#tab-non-ai")
Result: Tab switched successfully.
Thought: 接著我在輸入框中填入原始訊息文宣...
Tool Call: input_text("#input-message", "大促銷開始了！...")

給學員的教學思維

開發人員在導入 AI 系統時，應多加思考：「如何讓軟體本身擁有向人類『視覺化示範 AI 思考過程』的能力？」這不僅是優良的產品設計，也是將來自動化驗收與自動化運作 SOP 的核心組件。
